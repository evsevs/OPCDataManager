			ПЛАГИН OPC2OPC для DataManager.


Плагин OPC2OPC предоставляет доступ DataManager'a к данным ОРС серверов.
Поддерживаются стандарты OPC v.10a и OPC v.2.03.

В данной версии плагин поддерживает обну группу данных, т.е.
все данные, запрашиваемые из плагина по GetTags (передаваемые в плагин 
по SetTags) помещаются в группу. 
Первое обращение к данному тега обеспечивает добавление тега в группу, 
последующие обращения к тегу НЕ добавляют тег в группу.

!!!Первое обращение к тегам плагина проходит несколько медленнее последующих!!!

Подписка на нотификации поддерживается.

Чтение и запись тегов производится через интерфейс синхронного чтения (IOPCSyncIO).
Источником для тегов при чтении является - DEVICE.

Плагин использует тип данных, предоставляемый сервером-источником, т.е. тип
данных на сервере OPC и передаваемый в DM - ИДЕНТИЧНЫ.

Плагин имеет возможность подключения как к локальным ОРС серверам, так и к 
удаленным. Тип подключения задается в строке конфигурации плагина.

Конфигурация:
	Плагин конфигурируется строкой конфигурации, которая передается ему
	в качестве параметра метода Configure.

	Полный пример строки конфигурации можно получить от самого плагина по
	методу GetExampleCnfg. Она выглядит следующим образом:
	
	Пример:
	"OPCServerName_GUID_or_ProgID=OPCServer.MyServer;Computer_Name_or_IP=ASU-EROHIN;"
	либо
	"OPCServerName_GUID_or_ProgID={EE33DD30-74D9-11d5-9CB4-004005A3070A};Computer_Name_or_IP=1.1.1.1;"

	И в том и другом случае - строка верна.

	Для настройки контроля состояния источника применяются параметры:
	  ServerStatusControlPeriod - задающий период опроса состояния сервера в секундах, по
			умолчанию значение равно 10 секунд {"ServerStatusControlPeriod=10;"}. 
	  ResetErrorCount - количество ошибко при опросе состояния (количество периодов с неудачным
			результатом опроса), после которого производится удаление интерфейсов и производится 
			попытка соединения с сервером,  по умолчанию - 2 периода {"ResetErrorCount=2;"}.

	Для настройки параметров группы используются следующие параметры:
		GroupUpdateRate - (может не задаваться в строке)период обновления данных группы в ОРС сервере-источнике данных.
		По умолчанию период обновления данных равен 500 миллисекунд {GroupUpdateRate=500;}
		GroupReadSource - (может не задаваться в строке)параметр для указания источника данных для группы (кэш(CACHE=1) или устройство(DEVICE=2)).
		По умолчанию данные беруться из кэша {GroupReadSource=1}.
		

	При задании GUID и удаленного доступа плагин обратиться непосредственно по GUID
	сервера. При задании ProgID (если он не зарегистрирован локально) обращение пройдет через сервер
	OPCEnum (поставляется с версией 2.03).

	Параметр Computer_Name_or_IP можно опустить, если соединение будет производиться
	с локальным сервером (на тойже машине, где находится плагин).

	На стороне сервера ОРС (на удаленном компьютере) должна быть (по стандарту ОРС) 
	программа - СОМ сервер OPCEnum. При задании имени сервера в виде GUID наличие 
	OPCEnum не обязательно.
